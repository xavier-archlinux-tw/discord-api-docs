社群邀請透過兩個由 API 驅動的功能，擴展了 Discord 的邀請系統：`role_ids` 參數用於自動指派身分組，而 `target_users_file` 參數則用於限制哪些使用者可以接受邀請。

<Info>
如果您是伺服器管理員或社群經理，想在不撰寫程式碼的情況下建立身分組邀請，請參閱 [社群邀請指南](/developers/communities/guides/community-invites) 以了解 Discord UI 的工作歷程。
</Info>

本教學將引導您如何在機器人或應用程式中以程式設計方式使用社群邀請。學完本教學後，您將了解如何：

- 透過 API 建立可指派身分組的邀請
- 透過 API 為特定使用者群組建立目標式邀請

---

## 了解社群邀請功能

社群邀請透過兩個專為遊戲整合設計的強大功能，擴展了 Discord 現有的邀請系統：

### 身分組指派

使用 API 時，`role_ids` 參數可讓您指定一個或多個 Discord 身分組，當玩家接受邀請時，這些身分組將會自動指派。無需手動管理身分組。

**範例使用情境：**

- 授予身分組給支持者
- 授予與遊戲中角色或成就相關的身分組
- 授予具有頻道存取權限的身分組

<Info>
一旦使用者接受邀請並獲得身分組，即使邀請過期或被刪除，這些身分組仍會保留。您需要手動移除身分組（透過機器人或 Discord 介面）。
</Info>

### 目標式邀請

只有透過 Discord API 才能設定允許哪些使用者接受邀請。`target_users_file` 參數接受一個包含 Discord 使用者 ID 的 CSV 檔案。只有該清單中的使用者才能看到並接受邀請。

**範例使用情境：**

- 防止獨家成就的邀請分享（例如，為擁有 100% 完成度身分組的玩家建立一個獨特的邀請）
- 結合身分組以授予版主/管理員權限
- 結合身分組以授予付費支持者特殊身分組/頻道存取權

這些功能適用於所有標準邀請參數，例如 `max_age`、`max_uses` 等。

## 授予身分組邀請範例：自動指派玩家身分組

在此範例中，我們將展示如何使用 [頻道邀請 API](/developers/resources/channel#create-channel-invite) 作為邀請的一部分來授予身分組。我們將探討遊戲開發者如何將其遊戲連接回他們的 Discord 伺服器。

### 先決條件

在開始之前，請確保您已具備：

- 一個用於測試的 Discord 伺服器
- 您的帳戶已啟用開發者模式（以便您可以複製身分組、頻道和使用者 ID）
- 一個帶有機器人憑證的 Discord 應用程式（在 [開發者入口網站](https://discord.com/developers/applications) 中建立）
- 已安裝 Node.js，或您的機器人或應用程式能夠發出 HTTP 請求
- 機器人權限：
    - **建立即時邀請**（建立邀請所需）
    - **管理身分組**（使用 `role_ids` 自動指派身分組所需）

### 撰寫邀請程式碼

當玩家從您的遊戲加入您的 Discord 伺服器時，您可以授予他們 **玩家** 身分組，以存取遊戲專屬頻道。以下是如何建立一個自動執行此操作的邀請：

```javascript
// IMPORTANT: Never hardcode tokens or commit them to version control
// Use environment variables or a secure configuration management system
// Replace the placeholder below with your actual bot token, channel ID, and role ID
const BOT_TOKEN = 'REPLACE_WITH_YOUR_BOT_TOKEN_FROM_THE_DEV_PORTAL'; // 請替換為您從開發者入口網站取得的機器人憑證
const CHANNEL_ID = '1234567890123456789'; // 請替換為您的頻道 ID
const ROLE_ID = '9876543210987654321'; // 請替換為您的身分組 ID

const response = await fetch(
    `https://discord.com/api/v10/channels/${CHANNEL_ID}/invites`,
    {
        method: 'POST',
        headers: {
        'Authorization': `Bot ${BOT_TOKEN}`,
        'Content-Type': 'application/json'
        },
        body: JSON.stringify({
        role_ids: [ROLE_ID]
        })
    }
);

if (!response.ok) {
const error = await response.json().catch(() => ({
    message: `HTTP ${response.status}`
}));
throw new Error(
    `Failed to create invite: ${error.message}${error.code ? ` (code: ${error.code})` : ''}`
);
}

const invite = await response.json();
console.log(`Created invite: https://discord.gg/${invite.code}`); // 已建立邀請：
```

### 運作原理

1.  **`role_ids` 陣列**：包含在接受邀請時要指派的 Discord 身分組 ID（在此範例中我們只指派一個，但您可以使用此陣列指派任意數量的身分組）
2.  **回應**：傳回一個包含您可以分享的 `code` 的邀請物件

    </Step>

    </Step>

## 目標使用者範例：建立目標式支持者邀請

在此範例中，我們將展示如何使用 [頻道邀請 API](/developers/zh-tw/resources/channel#create-channel-invite) 僅允許特定使用者接受邀請。我們將探討遊戲開發者或實況主如何為其社群中的支持者授予特殊身分組和頻道。

### 先決條件

在開始之前，請確保您已具備：

- 一個用於測試的 Discord 伺服器
- 您的帳戶已啟用開發者模式（以便您可以複製身分組、頻道和使用者 ID）
- 一個帶有機器人憑證的 Discord 應用程式（在 [開發者入口網站](https://discord.com/developers/applications) 中建立）
- 已安裝 Node.js，或您的機器人或應用程式能夠發出 HTTP 請求
- 機器人權限：
    - **建立即時邀請**（建立邀請所需）
    - **管理伺服器**（使用 `target_users_file` 進行目標式邀請所需）
    - **管理身分組**（使用 `role_ids` 自動指派身分組所需）

### 撰寫邀請程式碼

如果您為您的遊戲或社群經營訂閱服務，您可以建立只有付費支持者才能接受的專屬邀請。這些身分組可以授予其他伺服器使用者無法看到的頻道存取權。以下是如何建立一個為特定使用者授予身分組的邀請：

```javascript
// 重要：切勿將憑證寫死在程式碼中或提交至版本控制系統。
// 請使用環境變數或安全的組態管理系統。
// 請將下方的預留位置替換為您實際的機器人憑證、頻道 ID 與身分組 ID。
const BOT_TOKEN = "REPLACE_WITH_YOUR_BOT_TOKEN_FROM_THE_DEV_PORTAL";
const SUPPORTER_CHANNEL_ID = '1234567890123456789';
const SUPPORTER_ROLE_ID = '1111222233334444555';

// 在生產環境中，您會從訂閱資料庫中獲取此資料
// 請將下方的範例使用者 ID 替換為您測試帳號的實際 Discord 使用者 ID
const activeSupporterIds = [
    '111111111111111111',
    '222222222222222222',
    '333333333333333333',
];

// 建立包含使用者 ID 的 CSV 內容
const csvContent = activeSupporterIds.join('\n');
const csvBlob = new Blob([csvContent], { type: 'text/csv' });

// 使用 FormData 進行 multipart/form-data 請求
const formData = new FormData();
formData.append('target_users_file', csvBlob, 'supporters.csv');

formData.append('payload_json', JSON.stringify({
    role_ids: [SUPPORTER_ROLE_ID]
}));

const response = await fetch(
`https://discord.com/api/v10/channels/${SUPPORTER_CHANNEL_ID}/invites`,
{
    method: 'POST',
    headers: {
    'Authorization': `Bot ${BOT_TOKEN}`
    },
    body: formData
}
);

if (!response.ok) {
    const error = await response.json().catch(() => ({
        message: `HTTP ${response.status}`
    }));

    let errorMessage = `建立邀請失敗: ${error.message || response.statusText}`;

    if (error.code) {
        errorMessage += ` (代碼: ${error.code})`;
    }

    if (error.errors) {
        errorMessage += '\n詳細錯誤：\n' + JSON.stringify(error.errors, null, 2);
    }

    throw new Error(errorMessage);
}

const invite = await response.json();

console.log(`將此連結分享給支持者：https://discord.gg/${invite.code}`);
```

### 運作原理

1. **`target_users_file`**：一個包含付費支持者 Discord 使用者 ID 的 CSV 檔案。只有此清單中的使用者才能看到並接受此邀請。
2. **`role_ids` 陣列**：自動指派「支持者」身分組，授予他們存取專屬頻道的權限。
3. **FormData**：上傳檔案時為必要欄位，其他參數則放入 `payload_json` 中。

<Info>
當僅使用如 `role_ids` 的 JSON 參數時，請使用 `Content-Type: application/json`。當上傳 `target_users_file` 時，您必須使用 `multipart/form-data`。
</Info>

### 設定支持者身分組

為了獲得最佳體驗，請為您的「支持者」身分組設定以下權限：

1. 在您的 Discord 伺服器中建立一個 **支持者** 身分組。
2. 建立一個私密頻道（例如 `#supporter-polls` 或 `#supporter-chat`）。
3. 右鍵點擊該頻道並點選 **「編輯頻道」**，以設定頻道權限，使只有擁有 **支持者** 身分組的使用者才能在其中檢視與傳送訊息：
    - 啟用該支持者專屬頻道的 **「檢視頻道」** 權限
    - 啟用 **「傳送訊息」** 與 **「新增反應」** 權限
