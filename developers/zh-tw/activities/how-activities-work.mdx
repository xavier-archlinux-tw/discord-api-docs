互動活動 (Activities) 是託管在 iframe 中，並在電腦版、行動版與網頁版 Discord 中執行的網頁應用程式。為實現此目的，我們使用 [postMessage](https://developer.mozilla.org/zh-TW/docs/Web/API/Window/postMessage) 協定在您的應用程式與 Discord 之間進行安全通訊。

[嵌入式 App SDK](https://github.com/discord/embedded-app-sdk) 藉由代您管理 `postMessage` 協定來簡化此流程。有關可用指令及其用法的詳細資訊，請參閱 [SDK 參考](/developers/zh-tw/developer-tools/embedded-app-sdk)。我們的 [範例專案](/developers/zh-tw/activities/overview#sample-projects) 提供了如何實作這些功能的實際範例。

## 啟動互動活動

在您的應用程式 [互動活動設定](https://discord.com/developers/applications/select/embedded/settings) 中啟用互動活動之後，您的 App 可以透過以下兩種方式啟動互動活動：
1. 当使用者在應用程式啟動器 (App Launcher) 中呼叫您 App 的進入點指令 (Entry Point Command) 時。
2. 透過回傳類型為 `LAUNCH_ACTIVITY` (類型為 `12`) 的互動回應。

## 專為單頁應用程式 (SPA) 設計

此 SDK 旨在供單頁應用程式 (Single-Page Application) 使用。我們理解開發者可能使用的框架或方法並不完全契合單頁應用程式。我們建議將這些框架巢狀嵌入在您互動活動的最上層單頁應用程式中，並依您的需要傳遞訊息。請參考 [巢狀框架 App](/developers/zh-tw/activities/overview#sample-projects) 範例專案以獲取關於此方法的引導。

## 互動活動生命週期

1. **初始化：** 當您的 iframe 被載入至 Discord 中時，其網址將會包含獨特的查詢參數。您的應用程式可以使用 Discord SDK 識別這些參數。
2. **握手流程：** 建立 SDK 執行個體會開始與 Discord 用戶端的握手流程。一旦連線建立，iframe 會收到 `[FRAME, {evt: 'READY', ...}]` 訊息。SDK 執行個體的 `ready()` 方法在成功建立連線後即會 resolve。
3. **授權與身分驗證：** 收到 `READY` 負載後，您的應用程式應執行授權與身分驗證，以獲取必要的權限 (Scopes)。這對於使用特定功能（例如 `rpc.activities.write`）至關重要。
4. **與 Discord 用戶端互動：** 驗證完成後，您的應用程式可以訂閱事件並向 Discord 用戶端發送指令。請注意，嘗試使用您獲得之權限範圍外的指令或事件將會導致錯誤。新增範圍可能會跳出 OAuth 授權視窗以要求使用者重新確認。
5. **中斷連線與錯誤：** 收到 `[CLOSE, {message: string, code: number}]` 訊息代表發生錯誤或需要重新啟動連線流程。
6. **發送錯誤或關閉請求：** 若要向 Discord 用戶端反映錯誤或要求關閉，請發送 `[CLOSE, {message?: string, code: number}]`。CLOSE_NORMAL 以外的代碼會向使用者顯示訊息，而 CLOSE_NORMAL 則會直接無聲關閉。

## 範例程式碼與互動活動生命週期圖

<Info>
以下是設定 SDK 的最小可行範例。請參閱我們的 [範例專案](/developers/zh-tw/activities/overview#sample-projects) 以獲取更完整的範例應用程式。
</Info>

```javascript
import {DiscordSDK} from '@discord/embedded-app-sdk';
const discordSdk = new DiscordSDK(YOUR_OAUTH2_CLIENT_ID);

async function setup() {
  // 等待來自 Discord 用戶端的 READY 載荷
  await discordSdk.ready();

  // 彈出 OAuth 授權視窗，並要求存取下方陣列中列出的權限範圍
  const {code} = await discordSdk.commands.authorize({
    client_id: YOUR_OAUTH2_CLIENT_ID,
    response_type: 'code',
    state: '',
    prompt: 'none',
    scope: ['identify'],
  });

  // 從您的後端應用程式伺服器獲取 access_token
  const response = await fetch('/.proxy/api/token', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      code,
    }),
  });
  const {access_token} = await response.json();

  // 使用 access_token 向 Discord 用戶端進行身分驗證
  auth = await discordSdk.commands.authenticate({
    access_token,
  });
}
```
