一個 **[Interaction (互動)](/developers/zh-tw/interactions/receiving-and-responding#interaction-object)** 是當使用者使用應用程式指令或訊息組件時，您的應用程式所接收到的訊息。

對於 [斜線指令 (Slash Commands)](/developers/zh-tw/interactions/application-commands#slash-commands)，它包含使用者提交的數值。

對於 [使用者指令 (User Commands)](/developers/zh-tw/interactions/application-commands#user-commands) 和 [訊息指令 (Message Commands)](/developers/zh-tw/interactions/application-commands#message-commands)，它包含執行該操作的目標使用者或訊息物件。

對於 [訊息組件 (Message Components)](/developers/zh-tw/components/reference)，它包含所使用組件的識別資訊。它還會包含一些有關如何觸發互動的中繼資料：`guild_id`、`channel`、`member` 以及其他欄位。您可以在下方的資料模型中找到所有的值。

### Interaction 物件 (Interaction Object)

###### Interaction 上下文類型 (Interaction Context Types)

可以使用或觸發互動的 Discord 上下文環境。有關應用程式指令使用互動上下文的詳細資訊，請參閱 [指令上下文說明文件](/developers/zh-tw/interactions/application-commands#interaction-contexts)。

| 名稱 (Name)     | 類型 (Type) | 說明 (Description)                                          |
|-----------------|-------------|-----------------------------------------------------------|
| GUILD           | 0           | 互動可以在伺服器中使用                                      |
| BOT_DM          | 1           | 互動可以在與應用程式 Bot 使用者的私訊中使用                 |
| PRIVATE_CHANNEL | 2           | 互動可以在群組私訊以及與應用程式 Bot 使用者之外的私訊中使用 |

<ManualAnchor id="interaction-object-authorizing-integration-owners-object" />
###### 授權整合服務擁有者物件 (Authorizing Integration Owners Object)

`authorizing_integration_owners` 欄位包含與互動相關之安裝授權使用者或伺服器的詳細資訊。對於安裝到使用者的應用程式，它可用於區分授權使用者與觸發互動的使用者（例如點擊訊息組件的使用者）。

僅在以下條件成立時，Key 值才會存在：
- 應用程式已被授權到與該 Key 值對應的[安裝上下文](/developers/zh-tw/resources/application#application-object-application-integration-types)（`GUILD_INSTALL` 或 `USER_INSTALL`）
- 該安裝上下文支援該互動在來源的 [互動上下文](/developers/zh-tw/interactions/receiving-and-responding#interaction-object-interaction-context-types)（`GUILD`、`BOT_DM` 或 `PRIVATE_CHANNEL`）中運作
- 對於指令觸發，該指令必須在安裝上下文中受支援（使用 [`integration_types`](/developers/zh-tw/interactions/application-commands#contexts)）

`authorizing_integration_owners` 中的值取決於 Key ——

- 如果 Key 是 `GUILD_INSTALL` (`"0"`)，其值取決於互動的來源：
    - 如果互動是在伺服器中觸發的，該值將是該伺服器的 ID
    - 如果互動是在與應用程式 Bot 使用者的私訊中觸發的，該值將是 `"0"`
- 如果 Key 是 `USER_INSTALL` (`"1"`)，該值將是授權使用者的 ID

###### 應用程式指令互動資料選項結構 (Application Command Interaction Data Option Structure)

所有選項都有名稱，且一個選項可以是一個參數和輸入值（在此情況下會設定 `value`），或者它可以表示一個子指令或群組（在此情況下它將包含一個最上層 Key 以及另一個 `options` 陣列）。

`value` 和 `options` 是互斥的。

| 欄位     | 類型                                                                                                                                                                                     | 說明                                                                                                                                        |
|----------|------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|-------------------------------------------------------------------------------------------------------------------------------------------|
| name     | string                                                                                                                                                                                   | 參數名稱                                                                                                                                    |
| type     | integer                                                                                                                                                                                  | [應用程式指令選項類型](/developers/zh-tw/interactions/application-commands#application-command-object-application-command-option-type) 的值 |
| value?   | string, integer, double, 或 boolean                                                                                                                                                      | 使用者輸入所產生的選項值                                                                                                                    |
| options? | array of [application command interaction data option](/developers/zh-tw/interactions/receiving-and-responding#interaction-object-application-command-interaction-data-option-structure) | 如果此選項是群組或子指令，則會存在                                                                                                           |
| focused? | boolean                                                                                                                                                                                  | 如果此選項是目前自動完成中焦點鎖定的選項，則為 `true`                                                                                        |

### Message Interaction 物件 (Message Interaction Object)

當訊息是回應一個沒有現有訊息的 Interaction 時，此物件會隨 [訊息物件 (message object)](/developers/zh-tw/resources/message#message-object) 一併發送。

<Info>
這意味著對 [訊息組件 (Message Components)](/developers/zh-tw/components/reference) 的回應不包含此屬性，而是包含一個[訊息參考 (message reference)](/developers/zh-tw/resources/message#message-reference-structure) 物件，因為組件*一律*存在於預先存在的訊息中。
</Info>

## 接收 Interaction (Receiving an Interaction)

當使用者與您的應用程式互動時，您的應用程式將會收到一個 **[Interaction](/developers/zh-tw/interactions/receiving-and-responding#interaction-object)**。您的應用程式可以透過以下兩種方式之一來接收互動：

-   透過 [Interaction Create](/developers/zh-tw/events/gateway-events#interaction-create) 閘道器事件
-   透過連出 Webhook

這兩種方法是**互斥的**；您*只能*透過其中一種方式接收互動。連接的用戶端可以處理 `INTERACTION_CREATE` [閘道器事件](/developers/zh-tw/events/gateway-events#interaction-create)，而下方詳述的 Webhook 方法則不需要連接的用戶端。

如果您希望透過基於 HTTP 的連出 Webhooks 接收互動，您必須為您的應用程式配置互動端點網址。您可以在互動總覽的[為互動做準備](/developers/zh-tw/interactions/overview#preparing-for-interactions)區段中閱讀有關為您的應用程式準備和新增互動端點網址的資訊。

### Interaction 中繼資料 (Interaction Metadata)

一個 [Interaction](/developers/zh-tw/interactions/receiving-and-responding#interaction-object) 包含中繼資料以協助您的應用程式進行處理，以及特定於互動類型的 `data`。您可以在其各自的頁面上找到每種互動類型的範例：

-   [斜線指令](/developers/zh-tw/interactions/application-commands#slash-commands-example-interaction)
-   [使用者指令](/developers/zh-tw/interactions/application-commands#user-commands-example-interaction)
-   [訊息指令](/developers/zh-tw/interactions/application-commands#message-commands-example-interaction)
-   [訊息組件](/developers/zh-tw/components/using-message-components)
-   [互動視窗組件](/developers/zh-tw/components/using-modal-components)

所有欄位的說明都可以在我們的[資料模型](/developers/zh-tw/interactions/receiving-and-responding#interaction-object)中找到。

現在您已從使用者獲取了資料，是時候對他們做出回應了。

## 回覆 Interaction (Responding to an Interaction)

互動的接收與回覆在底層都是 Webhooks。因此，回覆 Interaction 就如同發送 Webhook 請求一樣！

<Info>
互動回應具有與一般 HTTP API 請求相同的標頭要求。詳情請參閱[此處](/developers/zh-tw/reference#http-api)。
</Info>

您有多種方式可以回覆互動：

## Interaction 回呼 (Interaction Callback)

當回覆收到的互動時，您可以對 `/interactions/<interaction_id>/<interaction_token>/callback` 發送 `POST` 請求。`interaction_id` 是接收到的 payload 中該特定 Interaction 的唯一 ID。`interaction_token` 是接收到的 payload 中該互動的唯一 token。

如果您是透過閘道器接收 Interaction，您**必須透過 HTTP 做出回覆**。對 Interaction 的回覆**不會作為命令透過閘道器發送**。

**如果您對透過 HTTP 接收到的互動發送此請求，請以 202 且無 body 回應原始 HTTP 請求。**

```py
import requests

url = "https://discord.com/api/v10/interactions/<interaction_id>/<interaction_token>/callback"

json = {
    "type": 4,
    "data": {
        "content": "Congrats on sending your command!"
    }
}
r = requests.post(url, json=json)
```

<Info>
互動 `tokens` 的有效期限為 **15 分鐘**，可用於發送後續訊息，但您**必須在收到事件後的 3 秒內發送初始回應**。如果超過 3 秒的截止時間，token 將會失效。
</Info>

<Accordion title="內聯 HTTP 回應行為" icon="code">
    如果您透過 HTTP 接收互動，您的伺服器也可以回應接收到的 `POST` 請求。您會希望回應 `200` 狀態碼（如果一切順利），並指定一個 `type` 和 `data`，這是一個 [Interaction 回應](/developers/zh-tw/interactions/receiving-and-responding#interaction-response-object) 物件：

    ```py
    @app.route('/', methods=['POST'])
    def my_command():
        if request.json["type"] == 1:
            return jsonify({
                "type": 1
            })

        else:
            return jsonify({
                "type": 4,
                "data": {
                    "tts": False,
                    "content": "Congrats on sending your command!",
                    "embeds": [],
                    "allowed_mentions": { "parse": [] }
                }
            })
    ```
</Accordion>

## 後續訊息 (Followup Messages)

有時，您在回覆互動後想向使用者發送後續訊息。或者，您可能想編輯您的原始回應。不論您是透過閘道器還是連出 Webhook 接收 Interaction，您都可以使用以下端點來編輯您的初始回應或發送後續訊息：

-   [`PATCH /webhooks/<application_id>/<interaction_token>/messages/@original`](/developers/zh-tw/interactions/receiving-and-responding#edit-original-interaction-response) 用於編輯您對 Interaction 的初始回應
-   [`DELETE /webhooks/<application_id>/<interaction_token>/messages/@original`](/developers/zh-tw/interactions/receiving-and-responding#delete-original-interaction-response) 用於刪除您對 Interaction 的初始回應
-   [`POST /webhooks/<application_id>/<interaction_token>`](/developers/zh-tw/interactions/receiving-and-responding#create-followup-message) 用於發送新的後續訊息
-   [`PATCH /webhooks/<application_id>/<interaction_token>/messages/<message_id>`](/developers/zh-tw/interactions/receiving-and-responding#edit-followup-message) 用於編輯使用該 `token` 發送的訊息

<Info>
互動 Webhooks 與一般 Webhooks 共享相同的速率限制屬性。
</Info>

互動 tokens 的有效期限為 **15 分鐘**，代表您可以在該時間段內回應互動。

### 端點 (Endpoints)

<Info>
下方的端點不受應用程式的[全域速率限制](/developers/zh-tw/topics/rate-limits#global-rate-limit)約束。
</Info>

## 建立 Interaction 回應 (Create Interaction Response)
<Route method="POST">/interactions/[\{interaction.id\}](/developers/zh-tw/interactions/receiving-and-responding#interaction-object)/[\{interaction.token\}](/developers/zh-tw/interactions/receiving-and-responding#interaction-object)/callback</Route>

建立對 Interaction 的回應。Body 是 [interaction response](/developers/zh-tw/interactions/receiving-and-responding#interaction-response-object)。傳回 `204`，除非 `with_response` 被設定為 `true`，此時會傳回 `200`，且 body 為 [interaction callback response](/developers/zh-tw/interactions/receiving-and-responding#interaction-callback-interaction-callback-response-object)。

此端點也支持類似 Webhook 端點的檔案附件。上傳檔案以及 `multipart/form-data` 請求的詳細資訊請參閱[上傳檔案](/developers/zh-tw/reference#uploading-files)。

## 建立後續訊息 (Create Followup Message)
<Route method="POST">/webhooks/[\{application.id\}](/developers/zh-tw/resources/application#application-object)/[\{interaction.token\}](/developers/zh-tw/interactions/receiving-and-responding#interaction-object)</Route>

<Info>
如果互動是從使用者安裝型 App 發起的，且未安裝在伺服器中（代表[授權整合服務擁有者物件](/developers/zh-tw/interactions/receiving-and-responding#interaction-object-authorizing-integration-owners-object)僅包含 `USER_INSTALL`），則每次互動最多隻能發送 5 條後續訊息。
</Info>

為 Interaction 建立後續訊息。功能與[執行 Webhook](/developers/zh-tw/resources/webhook#execute-webhook)相同，但 `wait` 一律為 true。在使用此端點進行互動後續處理時，不支援 `thread_id`、`avatar_url` 和 `username` 參數。您可以使用 `EPHEMERAL` [訊息旗標](/developers/zh-tw/resources/message#message-object-message-flags) `1 << 6` (64) 來發送僅使用者可見的訊息。您也可以使用 `IS_COMPONENTS_V2` [訊息旗標](/developers/zh-tw/resources/message#message-object-message-flags) `1 << 15` (32768) 來發送基於[組件](/developers/zh-tw/components/reference)的訊息。

在使用 `DEFERRED_CHANNEL_MESSAGE_WITH_SOURCE` 回應互動後立即使用此端點時，此端點將發揮與[編輯原始互動回應](/developers/zh-tw/interactions/receiving-and-responding#edit-original-interaction-response)相同的功能以實現向後相容。在此情況下，不會建立新訊息，而是會編輯載入訊息。臨時 (ephemeral) 旗標將被忽略，並且您在初始延遲回應中提供的值將被保留，因為現有訊息的臨時狀態無法更改。此行為已被棄用，在此情況下您應改用[編輯原始互動回應](/developers/zh-tw/interactions/receiving-and-responding#edit-original-interaction-response)端點。
