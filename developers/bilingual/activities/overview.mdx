## Developing Activities

Whether you're developing a multiplayer game, a new social experience, or another creative idea, your Activity will be built as a web app that is run in an iframe in Discord on desktop, mobile, and web. You can get started by following the guide on [Building an Activity using the Embedded App SDK](/developers/en-us/activities/building-an-activity) or exploring the [sample projects](/developers/en-us/activities/overview#sample-projects)

The sections below provide an overview of the Embedded App SDK, but technical details about how Activities are run in Discord is covered in [How Activities Work](/developers/en-us/activities/how-activities-work).

### Embedded App SDK

The [Embedded App SDK](/developers/en-us/developer-tools/embedded-app-sdk) handles all of the communication between Discord and your app, making it easier to do common tasks like managing connected users, supporting mobile clients, and debugging your Activity.

The Embedded App SDK offers different events and commands to handle the communication between Discord and your Activity, which are covered more below.

#### Commands

The SDK has a set of commands you can call to interact with a Discord client, given you have the appropriate scopes. This is helpful when you want to do authorize and authenticate users, fetch information about your Activity, or update information for your Activity or an authenticated user.

Read the [Embedded App SDK documentation](/developers/en-us/developer-tools/embedded-app-sdk#sdk-commands) for a full list of commands, and details about each command.

#### Events

The SDK also has events you can subscribe (or unsubscribe) to for things happening in a connected client that are relevant to your Activity, given you have the appropriate scopes. This is helpful when you want to do things like handle initial connection and thrown errors, listen to updates about a connected user, and listen to events related to your Activity instance.

Read the [Embedded App SDK documentation](/developers/en-us/developer-tools/embedded-app-sdk#sdk-events) for a full list of events, and details for each event.

## 開發互動活動

無論您是要開發多人遊戲、新的社交體驗還是其他創意點子，您的互動活動都將被建置為一個在電腦版、行動版與網頁版 Discord 中透過 iframe 執行的網頁應用程式。您可以遵循 [使用嵌入式 App SDK 建置互動活動](/developers/zh-tw/activities/building-an-activity) 指南，或探索 [範例專案](/developers/zh-tw/activities/overview#sample-projects) 來開始。

下方章節提供嵌入式 App SDK 的概述，而有關互動活動如何在 Discord 中執行的技術細節則涵蓋於 [互動活動運作原理](/developers/zh-tw/activities/how-activities-work) 中。

### 嵌入式 App SDK (Embedded App SDK)

[嵌入式 App SDK](/developers/zh-tw/developer-tools/embedded-app-sdk) 處理 Discord 與您的 App 之間的所有通訊，讓您能更輕鬆地處理常見任務，例如管理已連線使用者、支援行動端用戶以及對您的互動活動進行偵錯。

嵌入式 App SDK 提供不同的事件與指令來處理 Discord 與您的互動活動之間的通訊，詳情將於下方進一步說明。

#### 指令 (Commands)

在您擁有適當權限 (Scope) 的前提下，SDK 提供一組您可以呼叫的指令來與 Discord 用戶端互動。當您想要授權並驗證使用者、獲取有關互動活動的資訊，或更新互動活動或已驗證使用者的資訊時，這會非常有用。

閱讀 [嵌入式 App SDK 說明文件](/developers/zh-tw/developer-tools/embedded-app-sdk#sdk-commands) 以獲取指令的完整列表以及每個指令的詳細資訊。

#### 事件 (Events)

在您擁有適當權限的前提下，SDK 也提供事件，讓您可以訂閱（或取消訂閱）已連線用戶端中與您的互動活動相關的活動。當您想要處理初始連線與拋出的錯誤、監聽已連線使用者的更新，以及監聽與您的互動活動執行個體相關的事件時，這會非常有用。

閱讀 [嵌入式 App SDK 說明文件](/developers/zh-tw/developer-tools/embedded-app-sdk#sdk-events) 以獲取事件的完整列表以及每個事件的詳細資訊。
