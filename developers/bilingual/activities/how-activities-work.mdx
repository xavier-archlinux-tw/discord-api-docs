Activities are web applications that run in an iframe within Discord on desktop, mobile and web. In order to achieve this, we use the [`postMessage`](https://developer.mozilla.org/en-US/docs/Web/API/Window/postMessage) protocol to enable secure communication between your application and Discord.

The [Embedded App SDK](https://github.com/discord/embedded-app-sdk) simplifies this process by managing the `postMessage` protocol on your behalf. For details on available commands and their usage, consult the [SDK Reference](/developers/en-us/developer-tools/embedded-app-sdk). Our [Sample Projects](/developers/en-us/activities/overview#sample-projects) provide practical examples of how to implement these features.

## Launching Activities

After you have Activities enabled in your Application's [Activity's settings](https://discord.com/developers/applications/select/embedded/settings), your app can launch Activities in two ways:
1. When a user invokes your app's Entry Point command in the App Launcher
2. By responding to an interaction with the `LAUNCH_ACTIVITY` callback type

Each of these are covered in more detail in the below sections.

### Entry Point Command

Activities are primarily opened when users invoke your app's [Entry Point command](/developers/en-us/interactions/application-commands#entry-point-commands) in the App Launcher.

When you enable Activities for your app, a [default Entry Point command](/developers/en-us/interactions/application-commands#default-entry-point-command) called "Launch" is created for you. By default, Discord automatically handles opening your Activity when your Entry Point command is run by a user.

Read more about setting up Entry Point commands in the [development guide](/developers/en-us/activities/development-guides/user-actions#setting-up-an-entry-point-command).

### Interaction Response

Activities can be launched in response to [command](/developers/en-us/interactions/overview#commands), [message component](/developers/en-us/interactions/overview#message-components), and [modal submission](/developers/en-us/interactions/overview#modals) interactions. To open an Activity, set the callback type to `LAUNCH_ACTIVITY` (type `12`) when [responding to the interaction](/developers/en-us/interactions/receiving-and-responding#responding-to-an-interaction).

## 啟動互動活動

在您的應用程式 [互動活動設定](https://discord.com/developers/applications/select/embedded/settings) 中啟用互動活動之後，您的 App 可以透過以下兩種方式啟動互動活動：
1. 當使用者在應用程式啟動器 (App Launcher) 中呼叫您 App 的進入點指令 (Entry Point Command) 時。
2. 透過回傳類型為 `LAUNCH_ACTIVITY` (類型為 `12`) 的互動回應。

下方的章節將針對這兩種方式提供更詳細的說明。

### 進入點指令 (Entry Point Command)

互動活動主要是當使用者在應用程式啟動器 (App Launcher) 中呼叫您 App 的 [進入點指令 (Entry Point Command)](/developers/zh-tw/interactions/application-commands#entry-point-commands) 時開啟。

當您為 App 啟用互動活動時，系統會自動為您建立一個名為「Launch」的 [預設進入點指令](/developers/zh-tw/interactions/application-commands#default-entry-point-command)。預設情況下，當使用者執行您的進入點指令時，Discord 會自動處理並開啟您的互動活動。

如需了解更多關於設定進入點指令的資訊，請參閱 [開發指南](/developers/zh-tw/activities/development-guides/user-actions#setting-up-an-entry-point-command)。

### 互動回應 (Interaction Response)

互動活動也可以在回應 [指令 (Command)](/developers/zh-tw/interactions/overview#commands)、[訊息元件 (Message Component)](/developers/zh-tw/interactions/overview#message-components) 與 [互動視窗提交 (Modal Submission)](/developers/zh-tw/interactions/overview#modals) 等互動時啟動。若要開啟互動活動，請在 [回應互動](/developers/zh-tw/interactions/receiving-and-responding#responding-to-an-interaction) 時，將回呼類型設定為 `LAUNCH_ACTIVITY`（類型 `12`）。

## Designed for Single-Page Apps (SPAs)

This SDK is intended for use by a single-page application. We recognize developers may be using frameworks or approaches that are not an exact fit for single-page applications. We recommend nesting those frameworks inside your Activity's top-level single-page application and passing messages as you see fit. Please refer to the [Nested Messages App](/developers/en-us/activities/overview#sample-projects) sample project for guidance on this approach.

## Activity Lifecycle

1. **Initialization:** When your iframe is loaded within Discord, it will include unique query parameters in its URL. These parameters are identifiable by your application using the Discord SDK.
2. **Handshake Process:** Constructing the SDK instance begins a handshake process with the Discord client. Once the connection is established, the iframe receives a `[FRAME, {evt: 'READY', ...}]` message. The `ready()` method of the SDK instance resolves once a successful connection has been established.
3. **Authorization and Authentication:** After receiving the `READY` payload, your application should perform authorization and authentication to acquire necessary permissions (scopes). This step is crucial for utilizing specific features or scopes, such as `rpc.activities.write`.
4. **Interacting with Discord Client:** Post-authentication, your application can subscribe to events and send commands to the Discord client. Note that attempting to use commands or subscribe to events outside your granted scope will result in errors. Adding new scopes may prompt an OAuth modal for user permission re-confirmation.
5. **Disconnection and Errors:** Receiving a `[CLOSE, {message: string, code: number}]` message indicates an error or a need to restart the connection process.
6. **Sending Errors or Close Requests:** To communicate an error or request a close from the Discord client, send `[CLOSE, {message?: string, code: number}]`. A code other than CLOSE_NORMAL will display the message to the user, while CLOSE_NORMAL results in a silent closure.

## 互動活動生命週期

1. **初始化：** 當您的 iframe 被載入至 Discord 中時，其網址將會包含獨特的查詢參數。您的應用程式可以使用 Discord SDK 識別這些參數。
2. **握手流程：** 建立 SDK 執行個體會開始與 Discord 用戶端的握手流程。一旦連線建立，iframe 會收到 `[FRAME, {evt: 'READY', ...}]` 訊息。SDK 執行個體的 `ready()` 方法在成功建立連線後即會 resolve。
3. **授權與身分驗證：** 收到 `READY` 負載後，您的應用程式應執行授權與身分驗證，以獲取必要的權限 (Scopes)。這對於使用特定功能（例如 `rpc.activities.write`）至關重要。
4. **與 Discord 用戶端互動：** 驗證完成後，您的應用程式可以訂閱事件並向 Discord 用戶端發送指令。請注意，嘗試使用您獲得之權限範圍外的指令或事件將會導致錯誤。新增範圍可能會跳出 OAuth 授權視窗以要求使用者重新確認。
5. **中斷連線與錯誤：** 收到 `[CLOSE, {message: string, code: number}]` 訊息代表發生錯誤或需要重新啟動連線流程。
6. **發送錯誤或關閉請求：** 若要向 Discord 用戶端反映錯誤或要求關閉，請發送 `[CLOSE, {message?: string, code: number}]`。CLOSE_NORMAL 以外的代碼會向使用者顯示訊息，而 CLOSE_NORMAL 則會直接無聲關閉。

```javascript
import {DiscordSDK} from '@discord/embedded-app-sdk';
const discordSdk = new DiscordSDK(YOUR_OAUTH2_CLIENT_ID);

async function setup() {
  // Wait for READY payload from the discord client
  await discordSdk.ready();

  // Pop open the OAuth permission modal and request for access to scopes listed in scope array below
  const {code} = await discordSdk.commands.authorize({
    client_id: YOUR_OAUTH2_CLIENT_ID,
    response_type: 'code',
    state: '',
    prompt: 'none',
    scope: ['identify'],
  });

  // Retrieve an access_token from your application's server
  const response = await fetch('/.proxy/api/token', {
    method: 'POST',
    headers: {
      'Content-Type': 'application/json',
    },
    body: JSON.stringify({
      code,
    }),
  });
  const {access_token} = await response.json();

  // Authenticate with Discord client (using the access_token)
  auth = await discordSdk.commands.authenticate({
    access_token,
  });
}
```

    }),
