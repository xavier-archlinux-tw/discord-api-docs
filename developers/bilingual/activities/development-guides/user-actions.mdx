## Open External Link

Since Activities are sandboxed, your app will need to perform a command in order for users to launch any external links. Users will be prompted inside Discord whether or not they want to open the external link.

#### Usage

```javascript
import {DiscordSDK} from '@discord/embedded-app-sdk';
const discordSdk = new DiscordSDK(clientId);
await discordSdk.ready();
// Once the sdk has established the connection with the discord client, external
// links can be launched
discordSdk.commands.openExternalLink({
  url: 'https://google.com',
});
```

#### User Experience

![external-link-modal](/images/activities/external-link-modal.png)

Users will see a modal inside the Discord app notifying them whether or not they want to proceed. By clicking **_Trust this Domain_**, users will not see a modal for that specific domain again.

## Open Invite Dialog

Getting an Application Channel Invite, as outlined in [these docs](/developers/en-us/resources/invite#get-invite), is not granted by any OAuth scopes. Nonetheless, the `openInviteDialog` command is available via the SDK. This command opens the Application Channel Invite UI within the discord client without requiring additional OAuth scopes.

This command returns an error when called from DM (Direct Message) contexts, so should only be called in Guild Voice or Text Channels. Similarly, this command returns an error if the user has invalid permissions for the channel, so using `getChannelPermissions` (requires OAuth scope `'guilds.members.read'`) is highly recommended.

#### Usage

```javascript
import {DiscordSDK, Permissions, PermissionUtils} from '@discord/embedded-app-sdk';
const discordSdk = new DiscordSDK(clientId);
await discordSdk.ready();

try {
  const {permissions} = await discordSdk.commands.getChannelPermissions();
  if (PermissionUtils.can(Permissions.CREATE_INSTANT_INVITE, permissions)) {
    await discordSdk.commands.openInviteDialog();
    // successfully opened dialog
  } else {
    console.warn('User does not have CREATE_INSTANT_INVITE permissions');
  }
} catch (err) {
  // failed to fetch permissions or open dialog
  console.warn(err.message);
}
```

#### User Experience

![Invite Dialog UI](/images/activities/invite-dialog.png)

Users will see a modal inside the Discord app allowing them to send an invite to a channel, friend, or copy an invite link to share manually.

## 開啟邀請視窗 (Open Invite Dialog)

在 SDK 中提供了 `openInviteDialog` 指令。此指令可在 Discord 用戶端內開啟應用程式頻道邀請 UI，而不需要額外的 OAuth 權限範圍，詳情請參閱 [這份說明文件](/developers/bilingual/resources/invite#get-invite)。

此指令在私訊 (DM) 上下文中呼叫時會返回錯誤，因此僅應在伺服器語音或文字頻道中呼叫。由於在使用者權限不足時此指令也會返回錯誤，因此高度建議先使用 `getChannelPermissions`（需要 OAuth 權限範圍 `'guilds.members.read'`）進行判斷。

#### 使用方式

```javascript
import {DiscordSDK, Permissions, PermissionUtils} from '@discord/embedded-app-sdk';
const discordSdk = new DiscordSDK(clientId);
await discordSdk.ready();

try {
  const {permissions} = await discordSdk.commands.getChannelPermissions();
  if (PermissionUtils.can(Permissions.CREATE_INSTANT_INVITE, permissions)) {
    await discordSdk.commands.openInviteDialog();
    // 成功開啟對話框
  } else {
    console.warn('使用者沒有 CREATE_INSTANT_INVITE 權限');
  }
} catch (err) {
  // 獲取權限或開啟對話框失敗
  console.warn(err.message);
}
```

## Open Share Moment Dialog

The easiest way for an application to share media to a channel or DM is to use the `openShareMomentDialog` command. This command accepts a Discord CDN `mediaUrl` (eg `https://cdn.discordapp.com/attachments/...`) and opens a dialog on the discord client that allows the user to select channels, DMs, and GDMs to share to. This requires no additional OAuth scopes, but does require the application to be authenticated.

Since `mediaUrl` must be a Discord CDN URL, it is encouraged to use the activities attachment API endpoint (`discord.com/api/applications/${applicationId}/attachment`) to create an ephemeral CDN URL. This endpoint accepts bearer tokens for any scopes, so it can be called from the application client using the authorized user's bearer token. The endpoint returns a serialized attachment, which includes a `url` attribute, which should then be passed to the DiscordSDK command as `mediaUrl`.

#### Usage

```javascript
import {discordSdk} from './wherever-you-initialize-your-sdk';
import {accessToken} from './wherever-you-store-your-access-token';

// some image
const imageURL = 'https://i.imgur.com/vaSWuKr.gif';

// get image data
const response = await fetch(imageURL);
const blob = await response.blob();
const mimeType = blob.type;

// image data as buffer
const buf = await blob.arrayBuffer();

// image as file
const imageFile = new File([buf], 'example.gif', {type: mimeType});

const body = new FormData();
body.append('file', imageFile);

const attachmentResponse = await fetch(`${env.discordAPI}/applications/${env.applicationId}/attachment`, {
  method: 'POST',
  headers: {
    Authorization: `Bearer ${accessToken}`,
  },
  body,
});
const attachmentJson = await attachmentResponse.json();

// mediaUrl is an ephemeral Discord CDN URL
const mediaUrl = attachmentJson.attachment.url;

// opens dialog in Discord client
await discordSdk.commands.openShareMomentDialog({mediaUrl});
```

## 開啟分享時刻視窗 (Open Share Moment Dialog)

應用程式若要將媒體分享到頻道或私訊 (DM)，最簡單的方法是使用 `openShareMomentDialog` 指令。此指令接受一個 Discord CDN 的 `mediaUrl`（例如 `https://cdn.discordapp.com/attachments/...`），並在 Discord 用戶端上開啟一個對話框，允許使用者選擇要分享的頻道、私訊和群組私訊 (GDM)。這不需要額外的 OAuth 權限範圍，但確實需要應用程式進行身分驗證。

由於 `mediaUrl` 必須是 Discord CDN 的網址，因此建議使用互動活動附件 API 端點 (`discord.com/api/applications/${applicationId}/attachment`) 來建立一個臨時的 CDN 網址。此端點接受任何權限範圍的 Bearer Token，因此可以使用已授權使用者的 Bearer Token 從應用程式用戶端進行呼叫。該端點會傳回一個序列化的附件物件，其中包含 `url` 屬性，接著應將其作為 `mediaUrl` 傳遞給 Discord SDK 指令。

#### 使用方式

```javascript
import {discordSdk} from './wherever-you-initialize-your-sdk';
import {accessToken} from './wherever-you-store-your-access-token';

// 某個圖像的網址
const imageURL = 'https://i.imgur.com/vaSWuKr.gif';

// 獲取圖像資料
const response = await fetch(imageURL);
const blob = await response.blob();
const mimeType = blob.type;

// 將圖像資料轉為 Buffer
const buf = await blob.arrayBuffer();

// 將圖像轉為 File 物件
const imageFile = new File([buf], 'example.gif', {type: mimeType});

const body = new FormData();
body.append('file', imageFile);

const attachmentResponse = await fetch(`${env.discordAPI}/applications/${env.applicationId}/attachment`, {
  method: 'POST',
  headers: {
    Authorization: `Bearer ${accessToken}`,
  },
  body,
});
const attachmentJson = await attachmentResponse.json();

// mediaUrl 是一個臨時的 Discord CDN 網址
const mediaUrl = attachmentJson.attachment.url;

// 在 Discord 用戶端中開啟對話框
await discordSdk.commands.openShareMomentDialog({mediaUrl});
```

## Setting Up an Entry Point Command

An [Entry Point command](/developers/en-us/interactions/application-commands#entry-point-commands) is required for users to be able to launch your Activity from the [App Launcher menu](https://support.discord.com/hc/articles/21334461140375-Using-Apps-on-Discord#h_01HRQSA6C8TRHS722P1H3HW1TV) in Discord.

When you enable Activities in your [app's settings](http://discord.com/developers/applications), a [default Entry Point command](/developers/en-us/interactions/application-commands#default-entry-point-command) is automatically created for your app. The default Entry Point command will use the `DISCORD_LAUNCH_ACTIVITY` (`2`) [handler type](/developers/en-us/interactions/application-commands#application-command-object-entry-point-command-handler-types), which means that Discord automatically launches your Activity for the user and posts a follow-up message into the channel where it was launched from.

If you want to handle sending messages yourself, you can update the handler to be `APP_HANDLER` (`1`). Details about Entry Point command handlers is in the [Entry Point command documentation](/developers/en-us/interactions/application-commands#entry-point-handlers).

#### Customizing the Default Entry Point Command

Entry Point commands can be customized in the same way as other [commands](/developers/en-us/interactions/application-commands). Since Entry Point commands can only be [global](/developers/en-us/interactions/application-commands#making-a-global-command), you'll use the HTTP endpoints for global commands:
- **Edit your existing Entry Point command's name or details** using the [Edit Global Application Command](/developers/en-us/interactions/application-commands#edit-global-application-command) endpoint. If you don't know the ID for your app's Entry Point command, use the [Get Global Application Commands](/developers/en-us/interactions/application-commands#get-global-application-commands) endpoint to retrieve it.
- **Make a different (option-less) command your Entry Point command** by updating its [command `type`](/developers/en-us/interactions/application-commands#application-command-object-application-command-types) to `PRIMARY_ENTRY_POINT` (type `4`). Your app can only have one Entry Point command, so if your app already has one, you must first [delete](/developers/en-us/interactions/application-commands#delete-global-application-command) it or [update](/developers/en-us/interactions/application-commands#edit-global-application-command) its [command `type`](/developers/en-us/interactions/application-commands#application-command-object-application-command-types).

#### Creating an Entry Point Command

To create a new Entry Point command, you can call the [Create Global Application Command](/developers/en-us/interactions/application-commands#create-global-application-command) endpoint and set the [command `type`](/developers/en-us/interactions/application-commands#application-command-object-application-command-types) to `PRIMARY_ENTRY_POINT` (type `4`).

Your command payload may look something like this:

```json
{
  "name": "launch",
  "description": "Launch Realms of Wumpus",
  // PRIMARY_ENTRY_POINT is type 4
  "type": 4,
  // DISCORD_LAUNCH_ACTIVITY is handler value 2
  "handler": 2,
  // integration_types and contexts define where your command can be used (see below)
  "integration_types": [0, 1],
  "contexts": [0, 1, 2]
}
```

In addition to the `type` and `handler` values, the command payload includes `integration_types` and `contexts` which let you configure when and where your command can be used:
- `integration_types` defines the [installation contexts](/developers/en-us/resources/application#installation-context) where your command is available (to a server, to a user's account, or both). If you don't set `integration_types` when creating a command, it will default to your app's [currently-supported installation contexts](/developers/en-us/resources/application#setting-supported-installation-contexts).
- `contexts` defines the [interaction contexts](/developers/en-us/interactions/receiving-and-responding#interaction-object-interaction-context-types) where a command can be run in Discord (in a server, in a DM with your app, and/or in DMs and Group DMs with other users).

Details about both of these fields are in the [command contexts](/developers/en-us/interactions/application-commands#contexts) documentation.

## 設定進入點指令 (Setting Up an Entry Point Command)

使用者需要透過 [進入點指令 (Entry Point command)](/developers/bilingual/interactions/application-commands#entry-point-commands) 才能從 Discord 的 [應用程式啟動器選單 (App Launcher menu)](https://support.discord.com/hc/articles/21334461140375-Using-Apps-on-Discord#h_01HRQSA6C8TRHS722P1H3HW1TV) 中啟動您的互動活動。

當您在 [應用程式設定](http://discord.com/developers/applications) 中啟用互動活動時，系統會自動為您的 App 建立一個 [預設進入點指令](/developers/bilingual/interactions/application-commands#default-entry-point-command)。預設進入點指令會使用 `DISCORD_LAUNCH_ACTIVITY`（`2`）[處理常式類型 (handler type)](/developers/bilingual/interactions/application-commands#application-command-object-entry-point-command-handler-types)，這代表 Discord 會自動為使用者啟動您的互動活動，並在該活動啟動的頻道中傳送一條後續追蹤訊息。

如果您想自行處理訊息傳送，可以將處理常式更新為 `APP_HANDLER`（`1`）。有關進入點指令處理常式的詳細細節，請參閱 [進入點指令說明文件](/developers/bilingual/interactions/application-commands#entry-point-handlers)。

#### 自訂預設進入點指令

進入點指令可以像其他 [指令 (Commands)](/developers/bilingual/interactions/application-commands) 一樣進行自訂。由於進入點指令只能是 [全域 (Global)](/developers/bilingual/interactions/application-commands#making-a-global-command) 的，因此您需要使用全域指令的 HTTP 端點：
- **編輯現有進入點指令的名稱或詳細資訊**：使用 [編輯全域應用程式指令](/developers/bilingual/interactions/application-commands#edit-global-application-command) 端點。如果您不知道您 App 進入點指令的 ID，請使用 [獲取全域應用程式指令](/developers/bilingual/interactions/application-commands#get-global-application-commands) 端點來獲取它。
- **將另一個（無選項的）指令設為您的進入點指令**：將其 [指令類型 `type`](/developers/bilingual/interactions/application-commands#application-command-object-application-command-types) 更新為 `PRIMARY_ENTRY_POINT`（類型 `4`）。您的 App 只能有一個進入點指令，因此如果您的 App 已經擁有一個，您必須先將其 [刪除](/developers/bilingual/interactions/application-commands#delete-global-application-command) 或 [更新](/developers/bilingual/interactions/application-commands#edit-global-application-command) 其 [指令類型 `type`](/developers/bilingual/interactions/application-commands#application-command-object-application-command-types)。

#### 建立進入點指令

若要建立新的進入點指令，您可以呼叫 [建立全域應用程式指令](/developers/bilingual/interactions/application-commands#create-global-application-command) 端點，並將 [指令類型 `type`](/developers/bilingual/interactions/application-commands#application-command-object-application-command-types) 設定為 `PRIMARY_ENTRY_POINT`（類型 `4`）。

您的指令 Payload 大致如下所示：

```json
{
  "name": "launch",
  "description": "啟動 Realms of Wumpus 互動活動",
  // PRIMARY_ENTRY_POINT 的類型值為 4
  "type": 4,
  // DISCORD_LAUNCH_ACTIVITY 的處理常式值為 2
  "handler": 2,
  // integration_types 與 contexts 定義了您的指令可以在哪裡使用（見下方說明）
  "integration_types": [0, 1],
  "contexts": [0, 1, 2]
}
```

除了 `type` 和 `handler` 值之外，指令 Payload 還包含了 `integration_types` 和 `contexts`，讓您設定在何時何地可以使用您的指令：
- `integration_types` 定義了您的指令可在哪些 [安裝上下文 (installation contexts)](/developers/bilingual/resources/application#installation-context) 中使用（伺服器安裝、使用者安裝，或兩者皆是）。如果您在建立指令時未設定 `integration_types`，它將預設為您 App [目前支援的安裝上下文](/developers/bilingual/resources/application#setting-supported-installation-contexts)。
- `contexts` 定義了指令可在 Discord 的哪些 [互動上下文 (interaction contexts)](/developers/bilingual/interactions/receiving-and-responding#interaction-object-interaction-context-types) 中執行（伺服器內、與您 App 的私訊中，以及/或者與其他使用者的私訊和群組私訊中）。

有關這兩個欄位的詳細資訊，請參閱 [指令上下文 (command contexts)](/developers/bilingual/interactions/application-commands#contexts) 說明文件。

## Encourage Hardware Acceleration

Activities that are compute intensive may benefit from encouraging users to enable hardware acceleration. When an application invokes the `encourageHardwareAcceleration` command the current status of the setting will be returned and the user will be prompted to update the setting, if applicable.

Users will see a modal inside the Discord app if Hardware Acceleration is disabled, encouraging them to change the setting. By clicking **Don't show me this again** they will not see the modal for _any application_ on this device again.

#### Best Practices

Switching the Hardware Acceleration setting causes the Discord client to quit and re-launch, so it is best practice to invoke this command as soon as possible, so users do not begin the experience of an application before restarting. Ideally, this is immediately after `await discordSdk.ready()`.

#### Usage

```javascript
import {DiscordSDK} from '@discord/embedded-app-sdk';
const discordSdk = new DiscordSDK(clientId);
await discordSdk.ready();
const {enabled} = await discordSdk.commands.encourageHardwareAcceleration();
console.log(`Hardware Acceleration is ${enabled === true ? 'enabled' : 'disabled'}`);
```

## 鼓勵啟用硬體加速 (Encourage Hardware Acceleration)

運算密集型的互動活動可能需要鼓勵使用者啟用硬體加速功能以獲得更好的體驗。當應用程式呼叫 `encourageHardwareAcceleration` 指令時，系統將會回傳該設定目前狀態的布林值，並在適當時提示使用者開啟該設定。

如果使用者的硬體加速功能已停用，他們將會在 Discord App 內看到一個提示視窗，鼓勵他們變更該設定。點擊 **「不要再顯示此訊息」** 後，使用者將不會在此裝置上的 _任何應用程式_ 中再次看到此提示。

#### 最佳做法

變更硬體加速設定會導致 Discord 用戶端關閉並重新啟動，因此最佳實踐是儘快呼叫此指令，以免使用者在重新啟動前已經開始體驗應用程式。理想情況下，這應該在 `await discordSdk.ready()` 之後立即執行。

#### 使用方式

```javascript
import {DiscordSDK} from '@discord/embedded-app-sdk';
const discordSdk = new DiscordSDK(clientId);
await discordSdk.ready();
const {enabled} = await discordSdk.commands.encourageHardwareAcceleration();
console.log(`硬體加速目前為 ${enabled === true ? '已啟用' : '已停用'}`);
```
